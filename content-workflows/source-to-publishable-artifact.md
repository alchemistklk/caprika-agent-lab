# Source To Publishable Artifact

## Capability

Turn source material into a reviewed public artifact without losing the evidence
trail.

## Workflow Boundary

- Input: source links, notes, screenshots, transcripts, generated media, or repo
  evidence.
- Tools: browser/research tools, file processors, image/video/audio generation,
  local editors, and publishing adapters.
- Files/artifacts: source notes, drafts, media candidates, review notes, final
  exports, and readback checks.
- Review gate: the creator checks factual accuracy, taste, private-detail
  boundaries, and platform fit before publishing.
- Output: a public post, article, video, image set, GitHub note, or reusable
  reference package.
- Recovery path: preserve intermediate artifacts so a failed generation or
  rejected draft can be revised without restarting from zero.

## Notes

This workflow is useful for documenting AI media work, agent product examples,
and public proof of work. It should stay evidence-backed: every public claim
should trace back to source material, an artifact, or a concrete implementation
record.
