# General Visual Style System

Use this file as a variable map before writing a poster or profile-header
prompt.

## Core Variables

```text
Project:
Role or creator identity:
Milestone or claim:
Audience:
Primary platform:
Main copy:
Secondary copy:
Metrics:
Brand palette:
Visual metaphor:
Objects and artifacts:
Forbidden elements:
```

## Palette

- Start with the actual product or creator palette.
- Pick one primary signal color and keep it disciplined.
- Use secondary accents sparingly for depth and rhythm.
- Avoid generic AI-purple unless it is truly part of the brand.
- Avoid one-note palettes that make the image feel flat.

## Typography

- Use strong, clean sans-serif typography for product or technical posters.
- Make the primary number or claim the dominant reading point.
- Keep labels minimal. Prefer data, role, or product signals over slogans.
- For profile headers, use less type than a poster. A short identity line is
  usually enough.

## Product Signals

Choose objects that match the project:

- Cursor forms
- Selection handles
- Product panels
- Canvas frames
- Grid surfaces
- Component blocks
- Workflow nodes
- Charts and metrics
- Local files, terminals, previews, or design artifacts

## Composition

- Poster: high hierarchy, strong center of gravity, readable feed-scale copy.
- Profile header: wide safe-crop layout, quiet avatar zone, visual energy away
  from the avatar.
- Use metaphor, but keep the real message readable.
- Avoid clutter, excessive lines, excessive arrows, and generic sci-fi.

## Example Preset: Open Design

- Base: deep black, charcoal, near-black gradients.
- Primary signal: neon green.
- Structure: off-white lines, panels, type, and geometric surfaces.
- Secondary accents: restrained cobalt/cyan blue, magenta, and warm orange.
- Signals: selection handles, cursor/pointer forms, product panels, canvas
  frames, grid surfaces, component blocks, agent workflow nodes.
