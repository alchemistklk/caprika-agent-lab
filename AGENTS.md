# Caprika Agent Lab Instructions

This repository is public-facing proof of work for Caprika's AI agent product
patterns.

Write saved artifacts in English by default.

## Boundaries

- Do not name private projects, customers, repositories, credentials, or
  unreleased business logic unless explicitly approved by the repository owner.
- Prefer public proof, sanitized case notes, and reusable patterns over resume
  claims.
- Keep examples small enough to inspect and run.
- Separate runnable experiments from pattern notes.

## Directory Rules

- `labs/`: runnable slices. Each lab should have a README, setup notes, and a
  clear expected output.
- `patterns/`: reusable engineering/product patterns extracted from real work.
- `case-notes/`: evidence-backed notes from public or approved projects.
- `content-workflows/`: source-to-artifact workflows and public content process
  notes.
- `templates/`: canonical formats for new entries.
- `assets/`: public-safe images and diagrams.

## Writing Style

- Start with the capability, then show the workflow boundary.
- Explain what is inspectable, recoverable, and reusable.
- Avoid vague claims such as "AI-powered" without implementation evidence.
- Keep private details abstracted into general patterns.
